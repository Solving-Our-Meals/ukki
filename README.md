## Introducing The Project Features (주요 기능)

### 회원가입
    * 약관 동의 및 회원가입을 할 수 있으며 이메일 인증이 필수로 동작해야 한다.
    * 회원가입 후 DB에 사용자 정보가 저장된다.
    * 유효성 검사 및 중복체크가 존재한다.
    
### 로그인
    * DB 내용을 토대로 아이디와 비밀번호를 비교해 유효성 검사 후 로그인을 한다.
    * 회원가입하고 로그인 한 인원은 JWT 토큰을 쿠키로 발급받는다.
    * 아이디 찾기가 가능하다.
    * 비밀번호 찾기가 가능하다.

### 식당 상세조회
    * 식당에 대한 혼밥난이도, 소개, 사진, 리뷰 확인 가능
    * 예약 가능 날짜 확인 (Default : Today)
    * 날짜를 선택하고 시간을 선택하는 순서를 가지며 예약을 마친다.
    * 예약을 마치면 안내문구가 나타나고 예약확인을 할 수 있다.
    * 예약 확인을 누르면 사용자의 정보, 예약정보가 예약 DB에 저장된다.

### 네비바
    * 기본적인 네비바의 기능들을 갖고 있다.
        1. 로고(홈)
        2. 게시판
        3. 마이페이지
        4. 로그인
        5. 회원가입

### 메인 페이지
    * 검색창에서 텍스트로 검색 가능
    * 제휴된 가게들 대표사진들과 상표명 나열 (배너식 슬라이드 방식)
    * 음식 종류별 카테고리를 제공 (클릭 시 위치기반 가까운 리스트 조회)
    * 자신의 위치를 누르면 가게의 위치가 표시된 지도 조회 가능
    * 사용자의 동의를 얻고(주의 문구) 위치 기반으로 랜덤한 식당을 자동 예약해주는 시스템 - 주의 문구에 취소 불가 및 바로 예약을 알림

### 플로팅 버튼
    * 주요 기능
    1. 홈
    2. 스크롤 탑
    3. 문의하기
        -> 문의하기에서 데이터베이스 기반 번호를 고르는 것에 따라 안내문구가 나타나고 자세한 문의 버튼을 넣어 1:1 문의로
            이동 가능 (예약, 환불, 기타 등)
            
### 문의 페이지
    * 상세한 문의를 할 수 있도록 문의 카테고리와 제목, 내용을 포함해 문의할 수 있는 페이지
        + 이미지 첨부 1개 가능
    * 각 카테고리별 기본 문의 양식은 다르게 나타남

### 마이 페이지
    * 회원 정보 수정 가능
    * 등급은 예약 횟수에 따라 분류되며 등급에 따라 뱃지 부여 및 혜택 제공
    * 자신이 문의 한 내용에 대한 정보 열람 가능

### 가게 페이지 (제휴 맺은 식당 주인 측)
    * 자신이 등록한 가게에 대한 정보 열람 가능
    * 시간별로 예약 가능 인원 조정 가능
    * 가게 예약 현황 확인 가능
    * 자신의 가게 리뷰에 대한 신고 가능(관리자 측에 알림)
    * 관리자에게 문의 가능

### 관리자 페이지
    * 식당 상세조회 가능
    * 식당에 대한 CRUD 가능
    * 리뷰에 대한 CRUD 가능
    * 회원에 대한 CRUD 가능
    * 가게를 선택해 해당 가게 실적 파일 출력 가능
    * 리뷰 신고에 대한 조회 및 삭제 가능

### QR Code
    * QR 코드를 이용해 예약 인증 가능
    * QR 코드를 이용해 포인트 사용 가능
